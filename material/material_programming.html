What is computer programming?

Computer programming, including web programming, is a process that starts with an original formulation of a problem and culminates in executable computer programs. The process consists of a set of activities:

  1. Understand the problem. (analysis)
  2. Plan the logic. (design)
  3. Code the program. (construct)
  4. Translate program into machine language.
  5. Test the program. (test)
  6. Put program into production.

What is problem solving?

Typical problems are complex. Many problems no longer appear as problems, because we have learned to solve them so well and so quickly that they take little time and little effort.

Break down problems and attempt to solve the pieces; attempt to solve each part. Even a simple problem needs to be planned, analysed, designed, constructed, and tested before it is implemented. What do you need to know?

  1. Describe the problem in own words
  2. State the OBJECTIVE of the problem we are trying to solve
  3. Identify the variables for INPUTs and OUTPUTs (ELEMENTS)
  4. What constraints need to be identified? (RELATIONS)
  5. Model the system: goal + elements + relations.

Use simple English (structured) or pseudocode. Include appropriate comments. Err on having too many comments. With practice you will learn to make sound and practical choices what constitutes too much and what is inadequate amount of comments.

Write out the algorithm. Keep track of constraints and assumptions. Look for improvements and alternatives, and revise.

Programming paradigms

Imperative programming A developer writes code that describes in exacting detail the steps that the computer must take to accomplish the goal.

A computer program is shaped by its data representation and the statements that determine its flow of control.

Every program (process) has input and output.

A programming paradigm is a style of programming. To program professionally, you need to learn either the object-oriented or functional programming paradigms.

State One of the fundamental differences among the various programming paradigms is the handling of state. State is the value of a program's variables while it is running. Global state is the value of a program's global variables while it is running.

Procedural programming paradigm Procedural programming paradigm is a programming style in which you write a sequence of steps moving toward a solution— with each step changing the program's state.You store data in global variables and manipulate it with functions

Functional programming paradigm Functional programming paradigm addresses the problems that arise in procedural programming by eliminating global state. A functional programmer relies on functions that do not use or change global state, the only state they use are the parameters you pass to the function. The result, a function returns, is usually passed on to another function. A functional programmer can thus avoid global state by passing it from function to function.

Object-oriented programming paradigm Object-oriented programming paradigm also addresses the problems that arise in procedural programming by eliminating global state, but instead of storing state in functions, it is stored in objects.

Understanding the problem

Even a simple problem needs to be planned, analysed, designed, constructed, and tested before it is implemented. Complex problems can be broken down into a set of simple problems.

Analyze the problem and consider alternate solutions, then select the most appropriate solution for a given situation.

  • Objective: what is the objective of the problem we are trying to solve?
  • Elements: what are the elements of the problem? (output and input)
  • Relations: what are the relations among the elements of the problem? (functionality, processes)
  • Algorithm: what is the step by step process that makes up the problem?
  • Model the system: prototype and test, identify areas of improvement, changes, alternatives, dependencies, constraints...
Design and implement the selected solution.

Start with organizing content with HTML5, style with CSS3, add behaviour with Javascript, as appropriate. Some notes:

  • Use PE approach to develop web apps.
  • Keep up-to-date with W3 Consortium. For example, we no longer include mime type in style and script elements.
  • Use the same convention as Java and Javascript for code style and documentation.
  • Validate HTML5 at validator.w3.org before adding any styles.
  • Validate CSS3 at jigsaw.w3.org before adding any Javascript code.
  • Organize Your Files
    • Create a project directory (folder) for every web site and every web application you are working on, usually under public_html directory.
    • Each project has its own set of sub-directories. The standard set includes the following sub-directories: css/, images/, media/, content/, scripts/
    • For larger projects, you may wish to use a sub-directory for each language used; for example, instead of creating scripts/ you may create js/, php/ and so on.
    • The project directory contains the home page index.html, the starting page of the web site or web app. All other HTML pages will be stored in the content/ directory.
  • Focus on the input and output of your web application.
  • Use "real" content where available.
  • Use "Lorem ipsum" dummy text and images, as place-holder, where the content is not available ahead of time or during development time.
  • Do not leave anything blank; do not use repetitious short phrases, or unrelated material as fillers.
  • Identify and document through comments, what styling and behaviour you intend to implement.
  • Do not forget to validate all code.

Standards, conventions, best practices.

Progressive Enhancement (wikiwand.com)

Progressive enhancement (PE) is a strategy for web design that emphasizes accessibility, semantic HTML markup, and external stylesheet and scripting technologies. PE consists of a set of core principles:

  • basic content should be accessible to all web browsers
  • basic functionality should be accessible to all web browsers
  • sparse, semantic markup contains all content
  • enhanced layout is provided by externally linked CSS
  • enhanced behavior is provided by unobtrusive, externally linked JavaScript
  • end-user web browser preferences are respected

The PE process consists of three steps:

  • Step One: Build solid foundation with HTML5. Identify content and use the most suitable (meaningful) markup to organize it. All content must be accessible without styles.
  • Step Two: Using well-formed and valid HTML5, style the elements for best presentation. All content must be accessible without scripts.
  • Step Three: Using well-formed and valid HTML5 styled with well-formed and valid CSS3, add behaviour with Javascript.
Writing code for readability

Writing code is similar to academic writing in that when you use or adapt code developed by someone else as part of your project, you must cite your source. However, instead of quoting or paraphrasing a source, you include an inline comment in the code. These comments not only ensure you are giving proper credit, but help with code understanding and debugging
