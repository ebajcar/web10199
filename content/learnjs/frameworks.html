Forms

An HTML form is a section of a document containing normal content, markup, special elements called controls (checkboxes, radio buttons, menus, etc.), and labels on those controls. Users generally "complete" a form by modifying its controls (entering text, selecting menu items, etc.), before submitting the form to an agent for processing (e.g., to a Web server, to a mail server, etc.)

How forms are processed

  1. value. Each element of a form can be dynamically assigned a value from JavaScript.
  2. id. If a form element contains an id attribute, that element can be referenced using the document.getElementById() method.
  3. id. A form element that does not contain an id attribute can be referenced using the document.forms[].elements[] arrays.
  4. name. The value assigned to the name attribute of a radio button group or checkbox button group is also the name of an array in which each button can be referenced by their index value.
  5. checkbox. When a form radio button or checkbox button has been selected its checked property becomes a true value.
  6. checked. A loop can be used to examine the checked property of each button in a button group to determine which are selected.
  7. options[]. The options of a selection list object are represented in the DOM by its options[] array
  8. selectedIndex. A selection list object has a selectedIndex property containing the array element index value of the currently-selected option
  9. selectedIndex. The value of the current selected option can be retrieved by specifying the selectedIndex value in the options[] array
  10. onchange. An event-handler function can be nominated by the onchange property to respond to changes in text field content
  11. onreset. An event-handler function can be nominated by the onreset property to respond when the user pushes a form’s reset button
  12. onsubmit. An event-handler function can be nominated by the onsubmit property to respond when the user pushes a submit button
  13. onsubmit. When the onsubmit event-handler returns a false value the form will not be submitted
  14. submit(). A form can be submitted from script by calling the submit() method of the form object

HTML5 and CSS3 form components

  • The HTML <form> element defines a form that is used to collect user input.
  • An HTML form contains form elements — different types of input elements: text fields, checkboxes, radio buttons, submit buttons,...
  • Form elements: form, input, textarea, label, fieldset, legend, select, optgroup, option, button, datalist, keygen, output
  • HTML input types: text, password, submit, reset, radio, checkbox, button
  • HTML5 added several new input types: color, date, datetime, email, datetime-local, month, number, range, search, tel, time, url, week
  • HTML5 attributes (Input Restrictions): disabled, maxlength, readonly, size, value
  • HTML5 added the following attributes for <input>: autocomplete, autofocus, form, height, and width, list, min, max, multiple, pattern, placeholder, required, step

Form validation

HTML5 makes available user interface elements that allow users to provide input. This ability to capture information from users is great, but you must make sure that user privacy and safety are protected as much as possible and that the website or web app cannot be exploited to disrupt the site's services. The objective is

  • to choose the correct user input controls (HTML5 input types), and
  • to set the appropriate attributes on those controls to ensure that the data is validated.

Data validation is the process of ensuring that computer input is clean, correct, and useful. Most often, the purpose of data validation is to ensure correct input to a computer application.

Validate data with built-in functions
  • HTML Input Attributes— disabled · max · min · pattern · required · type
  • CSS Pseudo Selectors— :disabled · :disabled · :optional · :required · :valid
  • DOM Properties validity— customError · patternMismatch · rangeOverflow · rangeUnderflow · stepMismatch · tooLong · typeMismatch · valueMissing · valid · validationMessage · willValidate
  • DOM Methods— checkValidity() setCustomValidity()

Data validation

There are two different types of form validation which you'll encounter on the web:

  • Client-side validation is validation that occurs in the browser, before the data has been submitted to the server. This is more user-friendly than server-side validation as it gives an instant response. This can be further subdivided:
    1. JavaScript validation is coded using JavaScript. It is completely customizable.
    2. Built-in form validation using HTML5 form validation features. This generally does not require JavaScript. Built-in form validation has better performance, but it is not as customizable as JavaScript.
  • Server-side validation is validation which occurs on the server, after the data has been submitted. Server-side code is used to validate the data before it is saved into the database. If the data fails authentication, a response is sent back to the client to tell the user what corrections to make. Server-side validation is not as user-friendly as client-side validation, as it does not provide errors until the entire form has been submitted. However, server-side validation is your application's last line of defense against incorrect or even malicious data. All popular server-side frameworks have features for validating and sanitizing data (making it safe).

In the real world, developers tend to use a combination of client-side and server-side validation.
