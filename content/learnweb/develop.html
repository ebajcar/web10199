Computer Programming

Programing in the narrow sense: By this, I refer to what many others call coding. Problems in this sense are problems of missing or broken software, and the solution is to write code that fixes them.

Programming in the wider sense: A fuller appreciation of programming acknowledges that coding is only part of the job. The larger job is to provide a solution that is complete, high-quality, and acceptable to the user.[2] This is much more than coding. It involves other activities, like requirements analysis, system design, or acceptability testing. It also includes lots of communication and collaboration, not just among the programming team but with the users too. Naturally, this requires skills beyond writing good code. [Bad Programming Practices 101]

Web programming, like computer programming is a process that starts with an original formulation of a problem and culminates in executable computer programs. Computer languages, like human languages, require you to use them in order to understand them. The process consists of a set of activities, just like solving a problem

  1. Understand the problem (ANALYSIS, PLAN)
  2. Plan the logic. (DESIGN, PLAN)
  3. Code the program. (referred to as CONSTRUCT, CODE, CREATE,…)
  4. Translate program into machine language. (COMPILE, INTERPRET)
  5. Test the program. (TEST, DEBUG)
  6. Put program into production. (DEPLOY, PUBLISH, IMPLEMENT,…)

Typical problems are complex. Many problems no longer appear as problems, because we have learned to solve them so well and so quickly that they take little time and little effort. All problems require due diligence to produce solutions of high quality.

Imperative programming

A developer writes code that describes in exacting detail the steps that the computer must take to accomplish the goal. A computer program is shaped by its data representation and the statements that determine its flow of control. Every program, process, has input and output.

Problem solving

Break down problems and attempt to solve the pieces; attempt to solve each part. Even a simple problem needs to be planned, analysed, designed, constructed, and tested before it is implemented. What do you need to know?

  1. Describe the problem in own words
  2. State the OBJECTIVE of the problem we are trying to solve
  3. Identify the variables for INPUTs and OUTPUTs (ELEMENTS)
  4. What constraints need to be identified? (RELATIONS)
  5. Model the system: goal + elements + relations.

Use simple English (structured) or pseudocode. Include appropriate comments. Err on having too many comments. With practice you will learn to make sound and practical choices what constitutes too much and what is inadequate amount of comments.

In general the vocabulary used in the pseudocode should be the vocabulary of the problem domain, not of the implementation domain.  The pseudocode is a narrative for someone who knows the requirements (problem domain) and is trying to learn how the solution is organized.  E.g., 

String.indexOf()

Pseudocode is a kind of structured english for describing algorithms. It allows the designer to focus on the logic of the algorithm without being distracted by details of language syntax.  At the same time, the pseudocode needs to be complete.  It describe the entire logic of the algorithm so that implementation becomes a rote mechanical task of translating line by line into source code.

Write out the algorithm. Keep track of constraints and assumptions. Look for improvements and alternatives, and revise.

Programming paradigms

A programming paradigm is a style of programming. To program professionally, you need to learn either the object-oriented or functional programming paradigms.

State, one of the fundamental differences among the various programming paradigms, is the handling of state. State is the value of a program's variables while it is running. Global state is the value of a program's global variables while it is running.

Procedural programming paradigm is a programming style in which you write a sequence of steps moving toward a solution— with each step changing the program's state.You store data in global variables and manipulate it with functions

Functional programming paradigm addresses the problems that arise in procedural programming by eliminating global state. A functional programmer relies on functions that do not use or change global state, the only state they use are the parameters you pass to the function. The result, a function returns, is usually passed on to another function. A functional programmer can thus avoid global state by passing it from function to function.

Object-oriented programming paradigm also addresses the problems that arise in procedural programming by eliminating global state, but instead of storing state in functions, it is stored in objects.
