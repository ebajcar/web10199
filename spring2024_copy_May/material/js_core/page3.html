Unobtrusive Javascript

Reminder: Adhere to Progressive Enhancement (PE) principles to create valid HTML5 and CSS3 documents before adding Javascript. Everything must be fully functional without Javascript.

Unobtrusive JavaScript is not a technique as such; unobtrusive JavaScript is not a matter of adding one makeUnobtrusive() function to your scripts in order to enter nirvana. Instead, it’s a way of thinking. To make sure that your scripts do not inconvenience anyone, you should make sure your scripts do not make any assumptions.

Assumption: everybody’s browser supports JavaScript.
Not true: In order to be unobtrusive to users, taking away the script should not preclude them from using your site, even though their interaction will be far less rich than that of users whose browser does support JavaScript.
Assumption: all browsers work the same.
Not true: In order to be unobtrusive to browsers, your script should steer clear of plain errors and compatibility problems, and take special devices such as speech browsers or mobile phones into account.
Assumption: everybody else will understand my code.
Not true: In order to be unobtrusive to fellow programmers, your script should consist of clear, clean code and feature a lot of comments to say what your code is (supposed to be) doing.

The principles to follow include:

  1. delay processing by including it at the end of body, after all marked up content.
  2. accelerate processing to the event DOMContentLoaded, before the window is loaded and rendering.
  3. avoid control boxes (alert, prompt,...), they pause all processing until the user dismisses them.
  4. avoid event attributes, you are mixing languages; hard to maintain.
  5. ultimately, move all Javascript to an external file(s).

Read further The principles of unobtrusive JavaScript on W3C

Delay code processing

Unobtrusive JavaScript places all script code in an external file and specifies a function to the DOMContentLoaded event to set behaviours when the HTML document has loaded.

Linking your Javascript file at the bottom of the document will let you control the rendering of the page better and can avoid some nasty performance problems with the way browsers render and execute Javascript.

<script src="js/script.js"></script>
</body>
</html>

Include scripts at the bottom of the body element. (Also see attributes defer and async for external Javascript files.)

By externally linking your CSS and Javascript files, you are allowing the browser to cache them in memory for each user. This improves the performance from the user's perspective by minimizing HTTP requests and downloading, hence speeding up the load time.

Accelerating Initialization

To ensure that the Javascript processing is done as soon as the Document Object Model (DOM) is ready to be access, use the event listener:

document.addEventListener( "DOMContentLoaded", initFunction, false )
