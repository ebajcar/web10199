Why is it important to write maintainable code?

Maintainable code means code is

  • readable, consistent, predictable, documented
  • looks as if it was written by the same person

As applications mature, your code needs to be reviewed, revised, and tweeked:

  • bugs are uncovered
  • new features are added
  • application needs to work in new environments
  • code gets re-purposed
  • code gets completely rewritten from scratch or ported to another architecture or another language

Few hours spent writing the code initially ends up in weeks spent reading it.

How do you organize files for your web projects?

It is important to follow instructions (in class and on the job). Misunderstanding and miscommunication may lead to a loss of marks on assignments, learning and applying wrong information, and a loss of credibility, time, and money on the job.

Use index0.html instead of index.html for all your web sites, unless (until) specifically asked otherwise.
