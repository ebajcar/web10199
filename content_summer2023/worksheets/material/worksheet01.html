Activity 1.3. STUDY Course material

1.3.1. Web programming, client/server environment, programming paradigms

—Study, make notes, then formulate your own answers. Collaborate through class discussions.

  1. What is full stack development? what technologies you need to know and understand?
  2. Describe the data flow in a web client/server environment.
  3. What are the differences between JavaScript running in the browser and language like PHP?
  4. Describe the process of designing client-server web apps. Compare to an essayist.
1.3.2. Unobtrusive Javascript
  1. delay processing — place at the end body
  2. accelerate processing — DOMContentLoaded event
  3. avoid control boxes (alert, prompt,...)
  4. avoid event attributes — hard to maintain
  5. ultimately, move code to an external file

—Study, make notes, then formulate your own answers. Collaborate through class discussions.

  1. Explain what is unobtrusive JavaScript. Describe at least four (4) ways how to make JavaScript unobtrusive.
  2. Where can JavaScript code be placed? where should it be placed?
  3. What are the eleven (11) common mistakes developers make in JavaScript?
  4. Explain the six (6) ways you can improve JavaScript performance?
  5. What is meant by imperative programming? Describe what JavaScript is
  6. Name two primary sources for JavaScript.
  7. What are the three ways to include JavaScript code into HTML document? Describe the pros and cons of using each.
  8. Explain why we delay the execution of JavaScript and then we accelerate the processing to a specific event.
  9. Explain why you would use a) document.write, b) console.log, c) alert dialogue, d) form element.
  10. Name at least three basic grammar rules for JavaScript statements.
  11. Explain how to debug JavaScript code.

Activity 1.4. PRACTICE problem solving

Create a project called mathClub. Follow Review and preparation. Implement each problem as a secondary page.

1.4.1. Add two numbers —Write a program that adds two numbers and displays result together with the two values provided by the user.
Potential solutions: Similar to Increment a number by 1 (class example). Need to add input field for the second number and make appropriate substitutions for the fixed number "1". You can skip some versions and leave the console.log calls for further debuggubg and testing. Make sure to validate HTML and CSS.

1.4.2. Guess a number —Write a program that will allow the user to guess a number between 1 and 10 that was randomly chosen by the program.
You probably have done this one in Java. Starting with pseudocode (language-gnostic), you can convert to any language. Note similarities and differences in the code.

1.4.3. Convert temperature —Write a program that calculates a given degrees Celsius in degrees Fahrenheit and displays a phrase that includes both values.

1.5.1. Review and preparation

Set up a project for each problem. Keep all versions of your solution for each project. For each problem, write out the algorithm that you plan to use (and modify if your implementation deviates from your original plan). Start with minimal HTML5 document and use the HTML comments to write out the algorithm, observations, and related info as you develop your solution. AVOID copy-and-paste, write comments in your own words, include links to related information, and type out the code out when you are copying.

  1. Work with your local files Use standard file organization to create project sub tree, public_html/syst10199/projectName Work on your local copy, then publish in the same sub-tree on dev.fast.sheridanc.on.ca
  2. Start with a minimal HTML5 document. create index0.html as your home page. (Do not use index.html to allow the instructor to view your directories on the server)
  3. Create secondary pages and update main navigation. create three secondary pages (stored in content folder), one for each exercise. Leave MAIN page area empty.
  4. Create secondary pages and update main navigation. update main navigation: Home will always take the user to the index page. All other links are set to # until we get the PHP and SQL in the second half of the course.
