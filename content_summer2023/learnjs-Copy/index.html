Including Javascript in HTML document

In most cases, Javascript is used to smooth out interactions, move elements around a page, and modify HTML. If you can make everything work without Javascript, you will not only have built-in fallbacks, if Javascript fails, but you will have to write far less Javascript to accomplish the same smooth experience for your users.

Like in CSS, with Javascript you target HTML elements and do stuff to them. With CSS, you apply styles; with Javascript, you apply behaviour. Like with CSS, there are three ways to include Javascript code into HTML documents:

  1. Inline Javascript using HTML event attributes... <button onclick="alert('inline js example'); "> ...
  2. Embedded Javascript using the <script> element ... <script>alert('embedded js example');</script> ...
  3. External Javascript (unobtrusive) using separate files... <script src="external.js"></script> ...

Inline Javascript using HTML event attributes

You can use Javascript in the opening HTML tags; for example, the event handler onMouseOver will trigger the alert event when the mouse moves over the link, as illustrated in Figure 1.

Inline JavaScript code can be assigned to any HTML event attribute, such as onload, or enclosed within a <script> element in the document body section â but is best avoided . All JavaScript code is best located in an external file whose path is specified to an src attribute of the <script> tag

Embedded Javascript using the <script> element

JavaScript code can be included in a web page by adding HTML <script> </script> tags, to enclose the script, and the opening tag must have a type attribute specifying the unique MIME type of text/javascript to identify the elementâs contents as JavaScript.


<script> 
  // code goes here
</script>

External Javascript (unobtrusive) using separate files

The separation of structure (HTML), presentation (CSS), and behaviour (JavaScript), is recommended by the W3 Consortium (W3C), and industry best practices, to make site maintenance simpler and each HTML document cleaner and easier to validate. Embedding an external JavaScript file in an HTML document requires a src attribute be added to the <script> tag to specify the path to the script file; typically JavaScript files are given a .js file extension. 


<script src="relative/path/local.js"></script>

The window.onload event can be used to set behaviours for older browsers when the entire page content has loaded. As there may be an undesirable initialization delay in waiting for the window.load event to be triggered modern web browsers have been made to recognize a DOMContentLoaded event that is triggered when just the HTML document and script have loaded â without waiting for external resources such as images.

Unobtrusive Javascript
Reminder: Adhere to Progressive Enhancement (PE) principles to create valid HTML5 and CSS3 documents before adding Javascript. Everything must be fully functional without Javascript.

Unobtrusive JavaScript is not a technique as such; unobtrusive JavaScript is not a matter of adding one makeUnobtrusive() function to your scripts in order to enter nirvana. Instead, itâs a way of thinking. To make sure that your scripts do not inconvenience anyone, you should make sure your scripts do not make any assumptions.

Assumption: everybodyâs browser supports JavaScript.
Not true: In order to be unobtrusive to users, taking away the script should not preclude them from using your site, even though their interaction will be far less rich than that of users whose browser does support JavaScript.
Assumption: all browsers work the same.
Not true: In order to be unobtrusive to browsers, your script should steer clear of plain errors and compatibility problems, and take special devices such as speech browsers or mobile phones into account.
Assumption: everybody else will understand my code.
Not true: In order to be unobtrusive to fellow programmers, your script should consist of clear, clean code and feature a lot of comments to say what your code is (supposed to be) doing.

The principles to follow include:

  1. delay processing by including it at the end of body, after all marked up content.
  2. accelerate processing to the event DOMContentLoaded, before the window is loaded and rendering.
  3. avoid control boxes (alert, prompt,...), they pause all processing until the user dismisses them.
  4. avoid event attributes, you are mixing languages; hard to maintain.
  5. ultimately, move all Javascript to an external file(s).

Read further The principles of unobtrusive JavaScript on W3C

Unobtrusive JavaScript places all script code in an external file and specifies a function to the DOMContentLoaded event to set behaviours when the HTML document has loaded.

Delay code processing

Linking your Javascript file at the bottom of the document will let you control the rendering of the page better and can avoid some nasty performance problems with the way browsers render and execute Javascript.

<script src="js/script.js"></script>

Include scripts at the bottom of the body element. (Also see attribute defer for external Javascript files.)

By externally linking your CSS and Javascript files, you are allowing the browser to cache them in memory for each user. This improves the performance from the user's perspective by minimizing HTTP requests and downloading, hence speeding up the load time.
