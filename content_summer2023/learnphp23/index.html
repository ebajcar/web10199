PHP Language

What distinguishes PHP from something like client-side JavaScript is that the code is executed on the server, generating HTML which is then sent to the client. The client would receive the results of running that script, but would not know what the underlying code was.
Introduction

PHP Hypertext Preprocessor (PHP) is an open source server-side scripting language designed for web develoment.

  • a PHP script is executed on the server, and the plain HTML result is sent back to the browser.
  • you need to have solid understanding of HTML, CSS, and Javascript. Progressive Enhancement approach applies.
    1. Organize with HTML5 → 2. Style with CSS3 → 3. Add behavior with JS → 4. Enhance with PHP
  • a PHP script processing instructions start with <?php and end with ?>
  • File extension for PHP files is .php

Coding and testing

We are treating the server as a development server (and production server) for the remainder of the course.

  • Continue working locally on your machine as before. The advantage is that you always have a copy of your project locally. The disadvantage is that you will need to publish, frequently, every time to test and debug.
  • Using online editor like the PHP Tryit Editor (w3schools) Most functionality can be tested by pasting your code in the editor, test and debug, then copying your code back into your file.
  • Using sFTP plugin in your editor See the NppFTP plugin link, how to work directly on the server using Notepad++. This is good if you are testing frequestly to debug your code. Don't forget to download the updated code to your local copy to keep them in sync. If you are using VS Code editor, you will need to find the equivalent plugin.
  • Another option is to download a web server, XAMPP. This is outside the scope of this course. One of the sites to explore, if interested, www.apachefriends.org. Keep in mind that you will need to publish your projects to (and recreate databases on) dev.fast.sheridanc.on.ca.
