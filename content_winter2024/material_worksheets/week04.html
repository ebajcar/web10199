Version 1
  1. Make sure your local copy is complete and tested.
  2. Is your site published? in projects directory?
  3. Are your project files organized and named properly? standard organization? (content, css, images, js), file naming conventions? meaningful names? home page named index0.html? secondary pages in content directory?
  4. Make sure you keep your project files clean (remove any unused files and remove any extra files and directories that may have been added created by the tools you use (if you did not create it, before you publish!).
  5. Did you use Progressive enhancement (PE) approach?
  6. Did you test and validate all your HTML code? all warnings and errors removed?
    validator.w3.org
  7. Did you test and validate all your CSS code? all warnings and errors removed? no inline CSS, a single external CSS file organized and documented properly, applied to all pages.
    jigsaw.s3.org/css-validator
  8. Did you test your navigation? from every page to every other page
  9. Do you have sufficient documentation? prologue on every page, comments explaining why. Include notes on alternatives that you considered.
  10. Is all content accessible? all errors, contrast errors, and alerts removed?
    WAVE web accessibility evaluation tool
  11. Is your design responsive? (does not matter for this version, but look forward!) Even for this version, do not use the entire width of the screen for your content, use only 60-80% only and make the right and left margins equal.Remainder: HEADER, FOOTER, and NAV are common page areas.
  12. Is all JS functionality working? (console output only)
    Increment a number (increment.html), Add two numbers (add.html), Guess a number (guess.html), Convert temperature (convert.html), Fibonacci numbers (fib.html), Area of a circle (circle.html)
Version 8
  1. Make sure your local copy is complete and tested.
  2. Is your site published? in projects directory?
  3. Evaluate your work by asking yourself questions 3 to 10 from Version 1 above.
  4. Is your design responsive?
    • use flexbox to style NAV as a single row for large screens and a single column for small screens
    • use flexbox to style MAIN that has three children, ASIDE, SECTION, ASIDE, in a row for large screens and in a column for small screens (generally, you can leave the ASIDEs blank)
  5. Is all JS functionality working?
    Increment a number (increment.html), Add two numbers (add.html), Guess a number (guess.html), Convert temperature (convert.html), Fibonacci numbers (fib.html), Area of a circle (circle.html)
    • JS code is embedded properly in the HTML document (internal)
    • using event methods (not event attributes)
    • getting value from user with <input type="number"> with appropriate constraints
    • using functions to make code reusable
    • consistent code style and sufficient useful comments; include notes on testing (how did you test it)
