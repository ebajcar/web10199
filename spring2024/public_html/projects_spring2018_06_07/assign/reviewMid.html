Functions, Events and the BOM

  • Functions use the keyword "function" followed by the name of the function, followed by a pair of brackets that may or may not contain parameters. The statements to be executed are enclosed within a pair of braces. A function can return a value.
  • object.innerHTML vs. document.write vs. createElement and appendChild methods
  • Get information about events with window.events
  • Get information about window properties
  • window.open and window.close
  • This is a event-driven environment.
  • Most events occur when a user causes an event, such as click, mouse over, presses a key, etc.
  • Some events are not caused by the user, such as when a page finished loading
  • there are three ways to act upon events 1. use event attributes, 2. use event methods, and use event listeners
  • event listeners are non-obtrusive and give most flexibility
  • have a closer read:
    • addEventListener on w3schools.com
    • removeEventListener on w3schools.com
    • how to remove event listener with anonymouse function on stackoverflow.com

DOM and objects

  • object-based environment; all elements, attributes, content,... are nodes in the DOM
  • each object has its own properties and methods
  • getElement methods can be used to get a handle on any available elements
  • parentElement.getElementById("idName") generates a handle to one unique element
  • parentElement.getElementsByTagName("tagName") generates an array of handles to zero, one, or more elements that have "tagName" within the scope of the parentElement
