Assignment 7 instructions

1. Objective

  • Create a members-only web site that provides members with an opportunity to play a variety of games.

2. Home page

  1. Use the template as specified in Assignment 4 to create the single-page site. Make sure that all obtrusive Javascript is replaced with external Javascript (js/main.js). Make sure that all inline styles are replaced with external styles file (css/main.js). Note keep the home page files separate from the games.
  2. An updated version of the template used in Assignment 4 is available. All files have been placed in a single directory; you will need to organize appropriately. The code has not been validated! Make sure you validate everything and often. Also, make sure you prettify (format) and comment code appropriately
  3. The template uses images bg3.jpg, canvas logo (js file),
  4. Evaluate carefully what code to leave internal or make external.
  5. If you are using your own Assignment 4, please make sure that you make all changes to match the layout and mechanics of the provided solution.
  6. Watch the page to page transitions here.
  7. Once you have the template working, copy it to index.php. Keep the template.html as a backup in case you need to revert to an earlier copy.

Note when the user clicks on it, it opens in the same window. The close button reloads the index page in the same window.
  3. Ultimate Tic Tac Toe (JS) Apply the games styles to your ultimate tic tac toe game. Make sure you use an unobtrusive Javascript version. Link to the second image in the first row, replacing the anyGame.html. Note when the user clicks on it, it opens in the same window. The close button reloads the index page in the same window.
  4. Classic Rock Paper Scissors (JS) Apply the games styles to your rock paper scissors game (Javascript version). Make sure you use an unobtrusive Javascript version. Link to the third image in the first row, replacing the anyGame.html. Note when the user clicks on it, it opens in the same window. The close button reloads the index page in the same window.

4. Form pages

  1. An updated version of the template used in Assignment 5 is available, see anyGame.html, which links in games.css.
  2. Login form Attach the login form page to the login button. Open the form in the same window. If the authentication is successful, present the home page with the LOGIN button displaying LOGOUT. When the user presses the LOGOUT button, log the user out and present the home page with the button displaying LOGIN. No other changes on the page at this time.
  3. BONUS: Modify the header to say “Welcome back, Firstname!” on successful login, and reverse both the tab and header on LOGOUT to the original content.
  4. Registration form Attach the registeration form page to the membership button. Open the form in the same window. Make sure you apply all possible css, js, and php validation to clean data before inserting the record into your database. On successful registration, advise the user that they can log in and take them to the home page. If unsuccessful, present the registration form if the user wishes to try again.
  5. Blog form Complete the blog form and incorporate on the site, attach to all fa-comments icons under each game. Open the form in the same window. Make sure you apply all possible css, js, and php validation to clean data before inserting the record into your database.
  6. BONUS: display two random blog entries in the BLOG POSTS area.

5. General

  1. For all games, use php to keep the <title> current.
  2. Add the last updated date on all pages in the footer using php.
  3. Makes sure to preserve the size of images and containers as in template. Size images to fit the smallest dimension (in this case, the height). If the contains are not the same height, consider using a link "...more" that will display the entire text in a modal.
